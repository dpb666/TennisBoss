KEY FEATURES:

    - Base: 25% conservative Kelly (0.04 * kelly_full)

    - Volatility: reduce in high-vol periods (baseline_vol / current_vol)
    - Drawdown: scale Kelly 100% → 25% as drawdown worsens
  • Position tracker: open positions, P&L tracking, correlation analysis
  • Hedge manager: calculate lay stakes for profit lock-in or risk reduction
  • Real-time P&L: track position value as odds move

API ENDPOINTS: 10+ endpoints
  ✓ GET  /v2/risk/portfolio-greeks        → Delta, Vega, Theta, correlation
  ✓ POST /v2/risk/update-bankroll         → Drawdown check + Kelly scale
  ✓ GET  /v2/risk/drawdown-status         → Current drawdown + recovery needed
  ✓ POST /v2/risk/volatility-normalize    → Stake scaling by surface vol
  ✓ POST /v2/risk/correlation-check       → New position correlation risk
  ✓ POST /v2/risk/correlation-matrix      → Full portfolio correlation matrix
  ✓ POST /v2/risk/cluster-detection       → Identify correlated player clusters
  ✓ POST /v2/risk/stress-test             → Full scenario analysis
  ✓ POST /v2/risk/var-cvar                → Value at Risk metrics
  ✓ GET  /v2/risk/portfolio-risk          → Comprehensive risk report
  ✓ GET  /v2/risk/risk-alerts             → Active risk alerts

KEY FEATURES:
  • Portfolio Greeks (adapted from options theory for betting):
    - Delta: net exposure to player outcomes (correlation-aware)
    - Vega: exposure to confidence changes (position-weighted std dev)
    - Theta: time decay (hours until match start)
    - Correlation: portfolio concentration estimate
    - Hedge ratio: optimal % to hedge for balance

  • Drawdown Alerts (4 tiers with auto-scaling):
    - Green: < 2.5% DD (Kelly = 1.0)
    - Yellow: 2.5%-5% DD (Kelly = 0.75)
    - Orange: 5%-7.5% DD (Kelly = 0.50)
    - Red: > 7.5% DD (Kelly = 0.25)

  • Volatility Normalization:
    - Surface-specific baselines (hard, clay, grass)
    - Inverse relationship: stake * (baseline_vol / current_vol)
    - Clamp: 50%-150% adjustment range

  • Correlation Matrix:
    - Build from ATP/WTA rankings (close ranks = correlated)
    - Cluster detection (find groups of highly-correlated players)
    - Reject new positions if correlation > threshold

  • Scenario Analysis:
    - Max loss (all bets lose)
    - Best case (all bets win)
    - Favorites lose (reversal scenario)
    - Odds collapse (20% adverse movement)
    - Confidence drop (15% confidence loss)
    - VaR/CVaR (95% confidence loss bounds)
    - Risk/reward ratio

ARCHITECTURE & DESIGN
================================================================================

LAYERED ARCHITECTURE:
  Layer 1: Analytics       (Edge detection, market analysis)
  Layer 2: Trading         (Automation, execution, positions)
  Layer 3: Risk            (Monitoring, controls, alerts)
  Layer 4: Realtime Engine (Settlement, WebSocket broadcasts)
  Layer 5: API             (REST endpoints, WebSocket)

DATA FLOW:
  Match → Analytics (EV, sharp signals)
       → Trading (auto-bet decision)
       → Position open
       → Live updates (P&L, hedges)
       → Risk (Greeks, drawdown, alerts)
       → Settlement
       → Calibration feedback

SEPARATION OF CONCERNS:
  • app/analytics/*  → Read-only market analysis
  • app/trading/*    → Execution & position state
  • app/risk/*       → Monitoring & controls
  • app/api/*        → HTTP/WebSocket interface
  • bot/realtime.py  → Async settlement & callbacks

NO MONOLITHIC COUPLING:
  • Each phase can evolve independently
  • Analytics works without trading
  • Trading works without risk (but recommended)
  • Risk works with any position set
  • Easy to add/modify/remove features
